"No repositories found" almost always means Netlify wasn't granted access to that specific repo during authorization. Fix it:

  1. In that Netlify repo-picker screen, look for a link like "Configure the Netlify app on GitHub" or "Can't see your repo? Configure on GitHub →" — click it.
  2. It opens GitHub's app-permissions page. Under Repository access, choose either "All repositories" or "Only select repositories" → and make sure maam-saab-website is selected.
  3. Click Save / Install.
  4. Go back to Netlify → the repo list will refresh and maam-saab-website will appear → pick it → Deploy.

If you still don't see it, the other common cause is the files were never pushed — check that github.com/Ma-am-Saab/maam-saab-website actually shows your files in the browser.
